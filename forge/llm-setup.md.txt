# FORGE — Local LLM Interface
## Production Plan for Claude Code

---

## Overview

Build a local web application called **FORGE** — a production-grade chat interface for Ollama with streaming, multi-model switching, and a full RAG pipeline. The user has an RTX 4090 with 24GB VRAM and Ollama installed locally.

**Goal:** A fast, beautiful, fully local app that talks to Ollama at `http://localhost:11434`. No cloud, no auth, no database — just a clean local tool.

---

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Framework | **Vite + React 18** | Fast dev server, HMR, small bundle |
| Language | **TypeScript** | Type safety for the RAG vector ops and API layer |
| Styling | **Tailwind CSS v3** | Utility-first, no runtime overhead |
| State | **Zustand** | Lightweight, no boilerplate |
| Markdown | **react-markdown + remark-gfm + rehype-highlight** | Full GFM, syntax highlighting |
| Syntax highlight | **highlight.js** | Works well with rehype-highlight |
| Icons | **lucide-react** | Clean, consistent |
| File parsing | Native browser FileReader + custom chunker | No deps needed |
| Vector math | Pure TypeScript (cosine similarity) | Vectors are float arrays, no lib needed |
| HTTP | Native `fetch` with ReadableStream | For Ollama streaming |

**No backend needed.** Everything runs in the browser against localhost:11434.

## Ollama API Layer (`src/lib/ollama.ts`)

Implement these functions:

### `fetchModels(): Promise<OllamaModel[]>`
- `GET http://localhost:11434/api/tags`
- Returns `data.models` sorted by name
- Throw with user-friendly message if offline

### `streamChat(params): AsyncGenerator<string>`
- `POST http://localhost:11434/api/chat`
- Body:
  ```ts
  {
    model: string,
    messages: {role, content}[],
    stream: true,
    options: {
      temperature, top_p, num_predict, repeat_penalty
    }
  }
  ```
- Read `response.body` as a `ReadableStream`
- Each newline-delimited JSON chunk: parse and yield `data.message.content`
- When `data.done === true`, stop
- Handle abort via `AbortController` — expose a cancel function

### `getEmbedding(text: string, model: string): Promise<number[]>`
- `POST http://localhost:11434/api/embeddings`
- Body: `{ model, prompt: text }`
- Returns `data.embedding`

### `checkHealth(): Promise<boolean>`
- `GET http://localhost:11434/` — returns true if reachable

## RAG Pipeline (`src/lib/rag.ts`)

### Chunking: `chunkText(text: string): string[]`
Strategy (in order of preference):
1. Split on double newlines into paragraphs
2. If paragraph count > 3, use **paragraph-based chunking** with merging:
   - Target chunk size: **600 chars**
   - Overlap: last **80 chars** of previous chunk prepended to next
   - Never split mid-sentence if avoidable
3. Fallback: **character-based** sliding window (size=600, overlap=80)
4. Filter out chunks < 30 chars

### Embedding: `embedDoc(doc: File, model: string, onProgress): Promise<RagDoc>`
- Read file as text
- Chunk it
- For each chunk, call `ollama.getEmbedding(chunk, model)`
- Call `onProgress(current, total)` after each embedding
- Return assembled `RagDoc`
- **Process one embedding at a time** (sequential, not parallel) to not overwhelm Ollama

### Vector Search: `searchDocs(query: string, docs: RagDoc[], model: string, topK: number): Promise<RagChunk[]>`
- Embed the query with `getEmbedding`
- Compute cosine similarity against all chunks across all docs
- Return top-K chunks sorted by score descending

### Cosine Similarity (`src/lib/utils.ts`): `cosineSim(a: number[], b: number[]): number`
```ts
export function cosineSim(a: number[], b: number[]): number {
  let dot = 0, na = 0, nb = 0;
  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i];
    na  += a[i] * a[i];
    nb  += b[i] * b[i];
  }
  return dot / (Math.sqrt(na) * Math.sqrt(nb) + 1e-10);
}
```

## Zustand Stores

### `modelStore.ts`
```ts
interface ModelStore {
  models: OllamaModel[];
  selectedModel: string;
  embedModel: string;        // '' means same as selectedModel
  status: OllamaStatus;
  statusText: string;
  loadModels: () => Promise<void>;
  setSelectedModel: (name: string) => void;
  setEmbedModel: (name: string) => void;
}
```

- `loadModels` fetches models, auto-selects first model, auto-prefers `nomic-embed-text` or `mxbai-embed-large` for embedModel if present
- `getActiveEmbedModel()` helper: returns `embedModel || selectedModel`

### `chatStore.ts`
```ts
interface ChatStore {
  messages: ChatMessage[];
  isGenerating: boolean;
  abortController: AbortController | null;
  addMessage: (msg: Omit<ChatMessage, 'id' | 'timestamp'>) => ChatMessage;
  updateLastAssistantMessage: (content: string) => void;
  clearChat: () => void;
  setGenerating: (v: boolean, abort?: AbortController) => void;
  cancelGeneration: () => void;
}
```

- Messages stored as array, IDs are `crypto.randomUUID()`
- `updateLastAssistantMessage` finds the last assistant message and updates its content (used during streaming)
- `cancelGeneration` calls `abortController.abort()`

### `ragStore.ts`
```ts
interface RagStore {
  docs: RagDoc[];

  addDoc: (doc: RagDoc) => void;
  removeDoc: (id: string) => void;
  toggleEnabled: () => void;
  clearDocs: () => void;
}
```

### `settingsStore.ts`
```ts
interface SettingsStore {
  params: GenerationParams;
  systemPrompt: string;
  updateParam: <K extends keyof GenerationParams>(key: K, value: GenerationParams[K]) => void;
  setSystemPrompt: (v: string) => void;

// defaults

**Persist settings** to `localStorage` using Zustand's `persist` middleware (settings and system prompt only — not messages or docs, those are session-only).

## Core Send Flow (`src/components/chat/InputArea.tsx` or a `useSendMessage` hook)

```
1.  Get userText from input, validate non-empty + model selected
2.  Add user ChatMessage to store
3.  If RAG enabled and docs.length > 0:
      a. Call ragStore.searchDocs(userText, docs, embedModel, topK)
      b. Build context string from top-K chunks
      c. Append to system prompt as "## Retrieved Context\n..."
      d. Store ragChunks for attaching to the assistant message
4.  Build messages array:
      [{role:'system', content: systemPrompt + ragContext}, ...history, {role:'user', content: userText}]
      Trim history to last 20 pairs (40 messages) to keep context manageable
5.  Create AbortController, call chatStore.setGenerating(true, controller)
6.  Add empty assistant ChatMessage to store (for streaming into)
7.  For each chunk from ollama.streamChat():
      a. Append to fullText
      b. Call chatStore.updateLastAssistantMessage(fullText)
8.  On done: attach ragChunks to the assistant message
9.  Finally: chatStore.setGenerating(false)
```

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    // Proxy Ollama to avoid CORS issues in dev

      '/ollama': {

        rewrite: path => path.replace(/^\/ollama/, ''),
        changeOrigin: true,
      }
    }
  }

```

> **Note:** With the proxy, use `/ollama` as the base URL in dev. But since the app targets localhost, direct `http://localhost:11434` calls also work fine. The proxy is optional insurance. If shipping as a static file (no dev server), use the direct URL.

## Error Handling

- **Ollama offline:** Show clear error in status bar + toast with instructions to run `ollama serve`
- **No models:** Toast with `ollama pull llama3.2` suggestion
- **Embedding failure:** Remove doc from list, show error toast with the specific error
- **Stream interrupted:** Mark message as `[Generation cancelled]` or show partial content as-is
- **Model not found during chat:** Show toast, don't crash
- **CORS errors:** If detected (network error on localhost), show specific guidance to set `OLLAMA_ORIGINS=*`

---

## Periodic Health Check

Poll `GET /api/tags` every 12 seconds when not generating.
- If previously offline and now online: reload models automatically
- If online → offline: update status bar dot to red
